# 题目
给定两个数组，编写一个函数来计算他们的交集
给定两个数组，编写一个函数来计算它们的交集。

示例 1:

输入: nums1 = [1,2,2,1], nums2 = [2,2]
输出: [2,2]
示例 2:

输入: nums1 = [4,9,5], nums2 = [9,4,9,8,4]
输出: [4,9]
说明：

输出结果中每个元素出现的次数，应与元素在两个数组中出现的次数一致。
我们可以不考虑输出结果的顺序。
进阶:

如果给定的数组已经排好序呢？你将如何优化你的算法？
如果 nums1 的大小比 nums2 小很多，哪种方法更优？
如果 nums2 的元素存储在磁盘上，磁盘内存是有限的，并且你不能一次加载所有的元素到内存中，你该怎么办？

# 思路
1. 哈希
   1. 哈希较小的那个数组 n
   2. 然后遍历 m （大数组），去 n 中找是否有对应，对应 --
   3. 复杂度 O(m+n), 空间 O(n)
2. 排序双指针
   1. 排序两个数组
   2. 指针指向两个数组的开头，并往右遍历
   3. 如果相等，则加入结果
   4. 如果不想等，指向小的数字的那个指针移位
   5. 复杂度 O(mlogm + nlogn), 空间 O(1) 保存答案不重要


这个问题的难度主要在进阶上面
- 如果给定的数组已经排好序
  - 双指针
- 如果 nums1 大小比 nums2 小很多
  - 用 nums1 进行hash
- 如果在磁盘上
  - 磁盘内存有限，不能一次性加载所有元素到内存，无法对 s2 进行高效地排序
  - 因此推荐使用方法一而不是方法2
  - 方法1只涉及到一部分的查询操作
